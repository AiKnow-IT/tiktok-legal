Security & data handling

This page explains what data AIKnow-IT processes, why we need it, and how users can revoke access. We follow a “minimum data” approach.

Data we process

  • TikTok OAuth tokens: access tokens (short-lived) and refresh tokens (to obtain new access tokens).
  • Posting metadata: caption/title, privacy settings, and basic publishing parameters.
  • Operational logs: limited technical logs (status codes, timestamps) for troubleshooting.

Why we process it

  • To connect a user’s TikTok account via OAuth (user consent).
  • To publish videos via TikTok Content Posting API (Direct Post).
  • To fetch publishing status and show success/failure information.

Where data is stored

Tokens and workflow configuration are stored in the environment running the automation (e.g., n8n credentials and any connected storage chosen by the user). We do not publish tokens publicly and recommend storing secrets in encrypted credential storage.

Retention

  • Tokens: stored as long as the user keeps the integration connected.
  • Logs: kept for a limited period (e.g., up to 30 days) to debug failures.

Revoking access

  1. Users can revoke TikTok access in TikTok account settings (connected apps).
  2. Users may also request deletion of stored tokens/logs by emailing support.

Support email: business.ai.gpt@gmail.com

Security measures

  • HTTPS for all web pages and API calls.
  • Least-privilege scopes (request only what is required, e.g., video.publish).
  • Restricted access to credentials and secrets.
  • Minimized logging of sensitive data (tokens are never intentionally logged).
